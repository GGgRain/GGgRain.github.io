DialogueInlineCommand

다이얼로그 인라인 커맨드는 [[다이얼로그 노드 베이스 콘텍스트]] 기반의 [[다이얼로그 노드]]의 콘텍스트 텍스트에 바인드 할 수 있는 호출 되는 로직을 담고 있는 에셋입니다.

작동 원리​

다이얼로그 인라인 커맨드는 다이얼로그 위젯이 텍스트를 업데이트하다가 바인드 된 커맨드가 있는 부분을 만나면 해당 커맨드를 실행하는 방식으로 실행됩니다.

[!Alert] [[인라인 커맨드 써보기]] 문서에 인라인 커맨드의 텍스트 바인드에 관련된 더욱 자세한 내용이 설명되어 있습니다.

다이얼로그 노드마다 사용 할 수 있는 인라인 커맨드가 다르며, 실행하려는 커맨드가 해당 노드에 가입 되어 있지 않다면 사용 할 수 없습니다. 프로젝트 세팅에서 노드 클래스마다 원하는 커맨드를 가입 시킬 수 있습니다.

[[다이얼로그 이벤트]]와의 차이점은 노드 인스턴스마다 하나의 독립적인 인스턴스로 붙는 다이얼로그 이벤트와 달리, 다이얼로그 인라인 커맨드는 노드의 클래스에 가입되는 방식이기에 단 하나의 인스턴스만을 공유하여 사용하기에 용량과 메모리 리소스를 더 적게 사용한다는 점과, 모든 데이터가 텍스트 안에 저장되므로 현지화시에 오류가 발생하거나 유실될 가능성이 존재한다는 점이 있습니다.

[!warning] 따라서 저희는 게임플레이에 큰 영향을 주는 로직은 다이얼로그 이벤트로 호출 할 것을 권장드리며, 특히 멀티플레이와 관련된 로직은 다이얼로그 이벤트로 호출 할 것을 권장 드립니다. 아예 커맨드를 해당 로직에서 배제해야한다면 이벤트 호출 커맨드 대신 다이얼로그 이벤트와 [[익스큐트 노드]]를 사용하는 것도 도움이 될 것입니다. 다이얼로그 인라인 커맨드 자체에는 되도록 게임에 영향을 끼치지 않는 비주얼 혹은 부가적인 기능을 사용하는 데에 사용 하십시오.

기본적으로 제공되는 다이얼로그 인라인 커맨드​

다음 인라인 커맨드들은 플러그인에 기본적으로 포함된 모든 종류의 [[다이얼로그 노드 베이스 콘텍스트]] 기반 다이얼로그 노드들에게 기본적으로 바인드 되어있습니다.

이름기본 시그니처어트리뷰트설명
DI_ChangeRenderTypeChangeRenderTypeFDialogueWidgetMode ModeChange the render type of the dialogue widget. This command will only work if the widget's base class is USpeechBubbleWidget_Handled.
DI_EnableSkipEnableSkipBoolean ValueChange the availability of skip action in the dialogue.
DI_EventEventFString KeyCall event assigned in the node for matching event key
DI_MoveToNextNodeMoveToNextNodeForce to move to the next node to play. If there is no node to play next, It will cause the dialogue to end.
DI_ResetSpeedResetSpeedSet the frequency of the text update to the default.
DI_SetSpeedSetSpeedfloat ValueSet the frequency of the text update.
DI_SkipSkipTry to skip the dialogue. It’s the same action that a player can perform by clicking the widget.
DI_SkipContextSkipContextTry to skip the dialogue’s text. It will not make the next dialogue node be played.
DI_StopDialogueStopDialogueBoolean ShouldImmediatelyRemoveRemove this dialogue widget from the screen.

다음 인라인 커맨드들은 기본적으로 특정 노드 클래스에만 바인드 되어있습니다. 대부분 특정 노드가 지원하는 특수한 기능 때문에 사용 됩니다.

셀렉트 노드​

이름기본 시그니처어트리뷰트설명
DI_StartSelectStartSelectCreate a set of buttons for the selection of the select node.

사용 방법​

다이얼로그 인라인 커맨드 만들기​

![[CreateInlineCommand.png]]

다이얼로그 매니저 에디터의 상단 툴바에서 Create New Inline Command 콤보버튼을 눌러서 새로운 인라인 커맨드를 만들 수 있습니다.

![[Image/GettingStarted/Untitled/override.png]]

그 후 ReceiveCommandTrigger 이벤트를 오버라이드 하여 해당 커맨드가 호출 되었을 때 실행할 로직을 작성 하실 수 있습니다.

![[Attributes.png]]

인라인 커맨드에 변수를 추가하여 퍼블릭으로 마킹할 경우 (혹은 인스턴스 편집가능으로 마킹할 경우) 해당 변수는 자동으로 해당 인라인 커맨드의 어트리뷰트가 됩니다. 어트리뷰트가 된 변수들은 커맨드 바인드 에디터에 표시 되어 에디터를 통해 값을 지정하거나 텍스트 상에서 값을 지정하여 사용할 수 있습니다. 어트리뷰트 변수들은 커맨드 실행 시에 바인드 시에 지정된 값으로 초기화 됩니다.

어트리뷰트를 초기화하지 않으면 (값을 지정하지 않으면) 클래스의 기본 값이 사용됩니다.

note

The inline commands support only the property types that can be stored in the string form and numeric form. The property types it supports are as following:

  • Boolean
  • Numeric types (int, float, double...)
  • Text (With a transient key and namespace. You can not localize this text.)
  • String, Name
  • Byte Types (Enums)

The property types it does not supports are as following:

  • Object, class reference
  • Map, Array, Set, Tuple, including all the other container classes.
  • Struct

As you can see, It has a certain limits on some applications, and this is way we also support Dialogue Events

![[Image/GettingStarted/Untitled/Editor.png]]

클래스 세팅에서 작성하신 Description과 퍼블릭으로 마킹 된 변수들의 Description은 커맨드 바인드 에디터상에서 해당 변수의 설명으로 표시됩니다. 이를 통해 좀 더 직관적이고 편리하게 커맨드를 사용 하실 수 있습니다.

다이얼로그 인라인 커맨드를 노드 클래스에 가입 시키기​

![[Assign.png]]

에디터의 프로젝트 세팅 탭에서 다이얼로그 인라인 커맨드 섹션에서 각 [[다이얼로그 노드]] 클래스마다 사용할 다이얼로그 인라인 커맨드 클래스를 가입 시킬 수 있습니다. 앞선 섹션에서 만든 다이얼로그 인라인 커맨드를 이 탭에서 원하는 노드 클래스에 원하는 시그니처로 가입 시킬 수 있습니다.

[!info] 커맨드 바인드 데이터는 프로젝트의 루트 디렉토리의 Config 폴더에 DefaultDialogueNodeCommandsRuntimeSetting.ini 파일에 저장됩니다.

이 때, 커맨드 바인드 엘리먼트들이 가지는 키 스트링 값을 시그니처라고 부릅니다. (예: DI_Event 인라인 커맨드의 시그니처 : Event)

이 시그니처는 Context Text 내부에서 해당 커맨드를 호출하는 데에 사용되므로 간결하고 커맨드의 기능을 잘 설명하는 것으로 선정하는 것이 좋습니다.

다이얼로그 인라인 커맨드를 바인드 하기​

이제 노드에 커맨드를 가입시키기까지 했으니, 커맨드를 사용할 일만 남았습니다.

커맨드를 호출 하기 위해 텍스트에 호출 심볼을 기입하는 것을 텍스트에 커맨드를 바인드 한다고 표현합니다. 이런 일련의 과정을 도와주는 아주 좋은 에디터가 이미 준비 되어있습니다. 자세한 내용은 인라인 커맨드 써보기 문서를 참고하십시오.

[!info] 이후에 설명 되는 내용은 고급 사용자를 위한 내용입니다. 일반적인 사용자 또한 해당 내용을 이해하고 있다면 매우 유용하나, 이미 다이얼로그 인라인 커맨드 바인드 에디터를 활용하여 더 쉬운 방법으로 커맨드를 바인드 할 수 있기 때문입니다. 해당 내용은 [[인라인 커맨드 써보기]] 문서를 참고하십시오.

다이얼로그 인라인 커맨드를 직접 바인드 하기​

앞서나온 시그니처를 Context Text에 [[시그니처]] 형태로 텍스트에 추가하여 텍스트의 특정 부분에 커맨드를 바인드 할 수 있습니다.

앞서 설명 했 듯 다이얼로그 위젯이 텍스트를 업데이트하다가 [[시그니처]] 형태의 텍스트를 만난다면 해당 시그니처에 해당하는 커맨드를 실행합니다.

어트리뷰트를 포함한 다이얼로그 인라인 커맨드를 직접 바인드 하기​

인라인 커맨드에는 시그니처 옆에 그 해당 커맨드가 실행될 때 필요한 어트리뷰트를 같이 기입할 수 있습니다. 쉽게 말해, 인라인 커맨드의 아규먼트입니다.

어트리뷰트의 이름을 기입하고 등호를 붙힌 뒤 큰따옴표 안에 값을 넣어줌으로서 활용이 가능합니다. 각각의 어트리뷰트는 띄어쓰기로 구분됩니다. 이런 조건을 고려하면 다음과 같이 여러가지의 어트리뷰트가 들어간 인라인 커맨드를 바인드 할 수 있습니다.

[[시그니처 어트리뷰트1이름="어트리뷰트1값" 어트리뷰트2이름="어트리뷰트2값" ...]]

마찬가지로 어트리뷰트를 초기화하지 않으면 클래스의 기본 값이 사용됩니다.
