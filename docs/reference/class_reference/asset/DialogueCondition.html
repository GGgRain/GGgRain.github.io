DialogueCondition

[!Info] 본 문서는 다이얼로그 컨디션의 기능과 구성에 대한 세부적인 설명을 포함합니다. 실제 사용 예시는 [[컨디션 써보기]] 문서를 참고하십시오.

다이얼로그 노드에 붙어 노드의 실행 가능성 여부를 표현할 때 사용되는 에셋입니다.

다이얼로그 노드는 다이얼로그 컨디션이 부착되어있을 시 해당 노드에 붙어 있는 모든 다이얼로그 컨디션이 테스트에서 통과 해야만 실행될 수 있습니다.

다이얼로그 컨디션 종류​

다이얼로그 컨디션은 두가지 종류가 있습니다.

  • 인스턴스드 다이얼로그 컨디션
  • 레벨 다이얼로그 컨디션

인스턴스드 다이얼로그 컨디션은 다이얼로그 매니저 상의 다이얼로그 노드 인스턴스에 인스턴스 형태로 부착 및 저장되는 컨디션입니다. 일반적으로 가장 많이 사용되는 형태입니다.

레벨 다이얼로그 컨디션는 말 그대로 액터로서 레벨에 배치가 가능한 형태의 컨디션입니다. 다이얼로그 매니저에서 이 컨디션을 가입시키면 그냥 그 컨디션 액터에 대한 소프트 액터 레퍼런스만을 가집니다. 월드 특정 위치, 혹은 월드 파티션등의 특수효과에 의한 영향을 주어야할 때 사용하기 좋습니다.

다이얼로그 컨디션 정의​

[!Info] 컨디션 에셋의 생성 및 사용에 관한 각 절차마다 세부적인 설명은 [[컨디션 써보기]] 문서를 참고 하십시오.

다이얼로그 컨디션 에셋을 생성 한 뒤에 그 에셋에 부여할 컨디션 로직을 작성하는 것은 다음 함수를 오버라이드 하는 것에서 시작됩니다 :

C++ 자손 클래스의 경우 : virtual bool CheckCondition(class UDialogueWidgetBase* Widget, const TMap<FName, UActorComponent*>& ConversationParticipants);

블루프린트 에셋인 경우 :

![[Image/GettingStarted/Untitled/override.png]]

여기서 원하는 로직에 따라 만약 컨디션 체크에 성공 했다면 true를, 아니라면 false를 리턴시키면 됩니다.

예제로 이런 방식으로 레벨에 있는 특정 액터가 파괴되지 않고 존재한다면 테스트를 통과시키는 DC_CheckTargetValid 에셋의 CheckCondition 로직입니다. :

![[bp.png]]

액터 레퍼런스가 유효하면 컨디션 체크에 성공하고, 유효하지 않으면 컨디션 체크에 실패하는 것을 볼 수 있습니다.

[!tip] 추가로, 다이얼로그 이벤트 클래스에 선언된 변수중 퍼블릭(인스턴스 편집가능)으로 마킹 된 프로퍼티는 다이얼로그 매니저 에디터 상의 디테일 탭에서 표시되며, 인스턴스 상에서 값을 변경 할 수 있게 됩니다.

다이얼로그 흐름 제어​

다이얼로그 컨디션만을 이용한 다이얼로그 흐름 제어​

다이얼로그 컨디션은 다이얼로그 전체의 노드 실행 흐름 제어에도 큰 영향을 미칩니다.

다이얼로그 노드간의 노드 우선도에 따라, 만약 한 아웃풋 핀에 여러개의 노드가 연결 되어있을 때, 플러그인은 우선도가 더 높은 순서대로 노드들을 실행시키려 합니다. 이때 더 높은 우선도를 가진 노드가 컨디션 체크에서 실패하여 실행되지 않는 것으로 결정 되면, 플러그인은 다음으로 높은 우선도를 가진 노드를 실행하려 시도합니다. 만약 모든 노드가 컨디션 체크에서 실패 한다면, 다음으로 실행 시킬 노드가 없으므로 다이얼로그는 종료됩니다.

[!Info] 노드 우선도와 관련된 설명은 [[다이얼로그 노드]] 문서를 참고 하십시오.

이 기능은 특히 어떤 노드가 특정한 상황에서만 재생되도록 하고 싶을 때 유용합니다. 가령 플레이어의 돈 혹은 특정 스탯이 일정 수치 이상일 때 특수한 대사를 출력하고 싶을 때 사용 할 수 있습니다. 그게 아니라면 게임 자체의 분기점처럼 작용 시킬 수도 있습니다.

브랜치 노드와 다이얼로그 컨디션을 활용한 다이얼로그 흐름 제어​

[[브랜치 노드]]와 다이얼로그 노드를 활용해서도 다이얼로그의 흐름을 제어 할 수 있습니다.

브랜치 노드는 브랜치 노드에 부착된 다이얼로그 컨디션의 컨디션 체크 테스트 성공 유무에 따라서, 테스트에 성공 했다면 True 핀에 연결된 노드들을, 실패했다면 False 핀에 연결 된 노드들을 실행시켜 줍니다.

용도는 위에서 설명한 다이얼로그 컨디션만을 이용한 다이얼로그 흐름 제어 섹션의 용도와 동일합니다.

부가적 요소​

FDialogueConditionStructure​

Simple Dialogue System 플러그인에서는 일반적으로 다이얼로그 컨디션의 집합을 FDialogueConditionStructure라는 이름의 스트럭쳐로 묶어서 처리합니다.

이 스트럭쳐는 간단하게 인스턴스드 컨디션과 레벨 컨디션을 담을 수 있는 배열과 두 배열 내의 모든 컨디션을 일괄적으로 테스트하는 함수로 구성된 랩퍼 스트럭쳐입니다.

실제로 다이얼로그 노드들은 따로 다이얼로그 컨디션을 위한 각각의 배열을 가지는게 아니라, 간단하게 이 FDialogueConditionStructure 스트럭쳐 하나를 가지며, 이 스트럭쳐의 컨디션을 체크 함 으로서 노드의 실행 여부를 결정합니다.

![[chekconcdition.png]]

또한 UDialogueFunctionLibrary::TestConditionStruct 함수를 이용해 블루프린트 상에서도 FDialogueConditionStructure의 컨디션을 체크 할 수 있습니다.
