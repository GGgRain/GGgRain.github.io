다이얼로그 프래그먼트 써보기

프래그먼트 시스템은 SimpleDialogueSystem의 가장 강력한 도구중 하나로, 각각에 노드에 미리 정의된 스토리지나 액션 모음인 [[다이얼로그 프래그먼트]]을 클릭 몇번으로 부착하여 [[다이얼로그 위젯 베이스]]의 동작과 데이터 그리고 [[다이얼로그 노드]]의 동작 혹은 데이터를 수정 할 수 있게 해주는 시스템입니다.

이 문서에서는 프래그먼트의 제작과 사용에 대한 모든 것을 다룹니다.

[!Info] 다이얼로그 프래그먼트의 클래스 구성에 대한 더욱 자세한 설명은 [[다이얼로그 프래그먼트]] 문서를 참고하십시오.

다이얼로그 프래그먼트 만들기​

우선, 첫번째 다이얼로그 프래그먼트를 만들어봅시다.

예제에서는 이 프래그먼트가 부착된 노드가 재생될 때 다이얼로그 위젯의 재생 속도를 원하는 값으로 변경하고나서 텍스트 데이터를 원하는대로 덮어 씌우는 프래그먼트를 만들어 보겠습니다.

이런 다이얼로그 프래그먼트를 만드는 이유는 재생 속도 조절로 다이얼로그 위젯와 상호작용하는 예제를 보여주고, 노드의 텍스트를 덮어 씌움으로서 다이얼로그 노드와 상호작용하는 방법을 보여드리기 위함에 있습니다.

아무 [[다이얼로그 매니저]] 에디터를 킨 후 툴바의 create new fragmemt 버튼을 클릭해봅시다.

그러면 새로운 프래그먼트 에셋이 생성됩니다. 이 프래그먼트를 더블클릭하여 열어봅시다.

우선 새롭게 적용할 속도를 나타낼 float 변수를 만들어 줍시다. 그리고 덮어씌울 텍스트 변수도 만들어줍시다.

프래그먼트 블루프린트 및 c++ 클래스 상에 퍼블릭으로(인스턴스 수정 가능으로) 마킹 된 변수들은 다이얼로그 매니저 에디터에서 자동으로 표시되고 수정 할 수 있습니다. 우리는 이 기능을 활용하여 에디터상에서 프래그먼트를 부착한 후, 원하는 속도를 넣어줄 것입니다.

우리가 다음에 해야 할 것은 몇가지 함수를 오버라이드하여 우리가 원하는 로직, 다이얼로그 위젯의 재생 속도를 조절하는 로직과 다이얼로그 노드의 텍스트를 변경하는 로직을 작성하는 것 입니다. ㅡㅡㅡㅡ 함수 설명 넣기

좋습니다. 이제 프래그먼트의 작성이 모두 끝났습니다.

노드에 프래그먼트 부착하기​

이제 다이얼로그 매니저로 돌아가 우리가 만든 프래그먼트를 부착할 노드를 찾이 우클릭 해줍니다.

Add fragment 섹션에서 우리가 방금 만든 프래그먼트를 찾아 선택해줍시다.

그러면 우리의 프래그먼트가 노드상에 붙은 것을 볼 수 있습니다. 이렇게 노드에 부착된 프래그먼트를 노드 프래그먼트라고 부릅니다. 자세한 내용은 [[다이얼로그 프래그먼트]]문서르 참고하십시오.

음, 그런데 여기 이 노드에 먼저 넣어진 텍스트가 너무 마음에 들어서 우리가 만든 프래그먼트가 텍스트를 덮어씌워버리면 좀 아쉬울 것 같습니다. 그래서 이 프래그먼트를 다른 노드에 옮겨 보고 싶습니다.

만약 이 프래그먼트를 다른 노드에 옮기고 싶다면 드래그 드랍을 통해 간편하게 옮길 수 있습니다.

만약 프래그먼트를 복사하고자 한다면 프래그먼트를 클릭하여 ctrl-c ctrl-v를 눌러 복사 할 수 있습니다.

이제 프래그먼트에 우리가 원하는 데이터를 기입하고, 한 번 실행시켜봅시다.

그러면 우리가 기입한 텍스트대로 텍스트가 덮어씌워지며 재생 속도도 변경 되는 것을 볼 수 있습니다.

노드에 붙은 프래그먼트 접근하기​

좋습니다. 그런데 만약, 우리가 다이얼로그 노드 내에서 이벤트 등을 통해 이 프래그먼트 레퍼런스에 접근 하고 싶다면 어떻게 해야할까요?

간단합니다. 노드 레퍼런스를 통해 FindFragmentByClass, FindAllFragmentByClass, GetAllFragment 함수를 통해 프래그먼트 레퍼런스를 받아 올 수 있습니다.

실제로 이벤트를 하나 만들어서 보여드리겠습니다.

이벤트에서 베이스 노드 레퍼런스로 우리가 방금 만든 프래그먼트를 가져와서 화면에 프래그먼트 오브젝트의 이름을 출력하게 만들겠습니다.

짠! 정상적으로 작동하는 걸 볼 수 읶습니다.

이런싣으로 프래그먼트에 접근하셔서 원하는 데이터를 수정 혹은 저장하거나, 원하는 함수를 호출하시면 됩니다.

다이얼로그 매니저에 프래그먼트 부착하기​

[!note] 여기까지 따라오셨다면 매우 잘 하셨습니다. 이제 다이얼로그 자체의 로직 제어에 관련된 시스템은 전부 다 둘러봤습니다.

[!note] 다음 튜토리얼은 매우 강력한 고급 내용을 다룹니다. [[기본 노드들 써보기]]에서 둘러보았던 노드들 외에도 여러분들이 원하는 대로 원하는 데이터와 원하는 로직을 지닌 다이얼로그 노드를 선언하여 사용하실 수 있습니다. [[커스텀 노드 만들어보기]] 튜토리얼으로 갑시다.
