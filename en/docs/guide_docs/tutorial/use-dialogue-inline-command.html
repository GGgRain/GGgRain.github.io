다이얼로그 인라인 커맨드 써보기

[!warning] 이 문서는 대략적인 다이얼로그 인라인 커맨드에 대한 설명을 포함합니다. 그러나 정확한 정보와 이해를 위해서는 [[다이얼로그 인라인 커맨드]] 문서를 참고하시는 것을 권장합니다.

[[다이얼로그 인라인 커맨드]]는 [[다이얼로그 노드 베이스 콘텍스트]]의 Context Text에 바인드하여서 원하는 액션을 원하는 텍스트의 한 지점에서 호출 할 수 있게 도와주는 강력한 도구입니다.

이 문서에서는 다이얼로그 인라인 커맨드를 어떻게 Context Text에 부착하여 (바인드하여) 활용하는 지를 다룹니다.

[!note] 해당 문서에서는 이미 만들어진 기본 제공 인라인 커맨드를 활용합니다. 커맨드 에셋 생성에 대한 자세한 내용은 [[다이얼로그 인라인 커맨드]] 문서를 참고하십시오.

[!note] 해당 문서에서는 이미 [[모놀로그 노드]]에 기본으로 등록된 다이얼로그 인라인 커맨드를 사용하여 노드 클래스에 커맨드를 가입 시키는 작업을 생략하였습니다. 기본 제공되는 인라인 커맨드가 아닌 새롭게 정의한 인라인 커맨드를 활용하기 위해선 커맨드를 노드 클래스에 가입 시켜야만 사용이 가능합니다. 자세한 내용은 [[다이얼로그 인라인 커맨드]] 문서를 참고하십시오.

[!note] 다이얼로그 인라인 커맨드 자체에 대한 정보는 [[다이얼로그 인라인 커맨드]] 문서를 참고하십시오. 이 문서는 다이얼로그 인라인 커맨드의 사용 및 활용 자체에 집중합니다.

커맨드 바인드 에디터를 통해 인라인 커맨드 바인드 하기​

[[다이얼로그 노드 베이스 콘텍스트]]의 노드 바디에서는 [[다이얼로그 인라인 커맨드]]의 바인드를 도와주는 편리한 버튼이 내장되어있습니다. 예제에서는 [[모놀로그 노드]]를 통해서 설명하겠습니다.

![[ContextNode.png]]

먼저, 노드의 Context Text에서 커맨드를 부착하고 싶은 부분을 클릭하여 텍스트 커서를 위치시키고, Bind Command 라고 쓰여져 있는 버튼을 눌러봅시다.

![[BindCommandMenu.png]]

그러면 바인드 할 인라인 커맨드를 선택 할 수 있는 메뉴가 나옵니다. 여러가지의 기본 제공 커맨드중 우리는 SetSpeed라고 써진 커맨드에 커서를 올려봅시다.

[!info] 각각의 커맨드에 대한 설명은 [[다이얼로그 인라인 커맨드]] 문서에 정리되어 있습니다.

![[CommandTooltip.png]]

그러면 이 SetSpeed 커맨드의 기능에 대한 대략적인 설명과 해당 커맨드가 사용하는 어트리뷰트 (매개 변수)에 대한 설명이 나옵니다.

보아하니, SetSpeed 커맨드는 다이얼로그 위젯의 텍스트 업데이트 속도를 변경 시키는 커맨드이고, Value 어트리뷰트에 들어간 값은 새롭게 적용할 텍스트 업데이트의 속도를 나타내는 것을 알 수 있습니다.

이제 이 SetSpeed를 눌러서 이 커맨드를 실제로 텍스트에 바인드 해봅시다.

![[binded.png]]

짠! SetSpeed 커맨드가 텍스트에 바인드가 된 것을 볼 수 있습니다.

바인드가 된 커맨드는 저렇게 까만 바탕에 커맨드의 순번이 들어간 모습의 버튼으로 표시가 되며, 텍스트 아래에는 각 순번에 대한 커맨드의 내용 전문을 표시해주는 표시 박스가 추가 되는 것을 볼 수 있습니다.

이렇게 하면 다이얼로그 위젯에서 텍스트가 업데이트가 되다가, 저기 Simple Di... 부분을 지날 때 우리가 바인드한 SetSpeed 커맨드가 호출이 됩니다. 매우 간단하지요?

그런데 위에서 보았듯, SetSpeed 커맨드는 Value라는 어트리뷰트를 사용하는데, 우리는 아직 이 어트리뷰트의 값을 초기화하지 않았습니다. 이제 이 값을 변경해봅시다.

[!info] 물론, Value 어트리뷰트를 초기화 하지 않더라도 엔진 크래쉬가 일어나는 등의 문제는 일어나지 않습니다. 초기화 되지 않은 어트리뷰트는 자동으로 클래스 디폴트 값으로 초기화 되기 때문입니다. 하지만 원하지 않는 동작이 일어날 태니 한번 수정 해 봅시다.

바인드 된 다이얼로그 인라인 커맨드 수정하기​

텍스트 위에 표시되어 있는 바인드 된 커맨드를 눌러 봅시다.

![[CommandEdit.png]]

그러면 다음과 같은 메뉴가 표시되는 것을 볼 수 있습니다.

이 메뉴는 커맨드 에디터입니다. 어트리뷰트를 포함한 커맨드의 모든 것을 수정 할 수 있습니다. 위에 표시가 된 Raw Command의 문자열을 수정함으로서 직접 수동으로 커맨드 자체를 수정하거나, 아예 다른 커맨드로 이 커맨드를 덮어 씌울 수 있습니다.

그러나 지금은 중앙 부분의 Value 어트리뷰트를 수정하는 섹션만 보겠습니다. Value 어트리뷰트 왼쪽에 있는 체크박스를 체크 해보십시오.

![[CommandAttributeEnabled.png]]

그러면 이제 Value 어트리뷰트의 값이 수정 가능하게 변한 것을 볼 수 있습니다. Raw Command 부분에 Value="" 라는 텍스트 또한 추가 된 것을 볼 수 있습니다.

방금 우리가 체크박스를 체크 함으로서 해당 어트리뷰트의 값을 초기화하겠다는 것을 명시한 것입니다. 이제 우리는 우리가 원하는 대로 저 값을 수정하여서 커맨드의 동작을 제어 할 수 있습니다.

값을 원하는 대로 수정 했다면 그래프의 빈 곳을 클릭하여 커맨드 에디터를 닫고 변경사항을 저장 할 수 있습니다.

이제 해당 다이얼로그 매니저를 실행해서 인라인 커맨드를 실행시켜보면 다이얼로그 위젯의 텍스트 업데이트 주기가 우리가 방금 설정한 대로 변하는 것을 볼 수 있습니다.

여기까지 오셨다면 잘 하셨습니다! 인라인 커맨드의 활용에 관한 기초적인 지식을 모두 터득하셨습니다.

[!note] 이벤트와 인라인 커맨드보다 더욱 다이얼로그를 위한 추가 데이터 저장 및 다이얼로그 변조 및 게임 플레이 로직 실행에 능한 멋쟁이가 있습니다. 다음 튜토리얼은 [[프래그먼트 시스템 써보기]] 입니다.

[!note] ......저희가 이렇게 까지 게임플레이 로직 실행을 도와주는 시스템을 다양하게 추가 한 것은 전부 이유가 있습니다. 다음 튜토리얼에서 명쾌하게 설명이 될 테니 조금만 더 끈기있게 따라와주시길 바랍니다.
