첫 다이얼로그 만들기

Simple Dialogue System을 엔진에 설치하고 프로젝트에서 활성화 했다면, 이제 첫 다이얼로그를 만들어 봅시다.

다이얼로그 매니저 만들기​

먼저 콘텐츠 브라우저에서 우클릭을 하여 뜨는 콘텍스트 메뉴에서 Dialogue -> Dialogue Manager 를 클릭하여 새로운 에셋을 생성합니다.

![[dmcreate.png]]

[[다이얼로그 매니저]] 에셋은 각종 다이얼로그 노드를 부착하여 자신만의 다이얼로그를 표현할 수 있는 스크립트 묶음입니다. Simple Dialogue System의 모든 다이얼로그들은 다이얼로그 매니저를 통해서 구현됩니다. 자세한 내용은 [[다이얼로그 매니저]] 문서를 참고하십시오.

![[Image/GettingStarted/Untitled/Editor.png]]

만들어진 다이얼로그 매니저를 더블 클릭하면 다이얼로그 매니저 에디터가 표시됩니다. 여기서 [[다이얼로그 노드]]를 이 다이얼로그 매니저 에셋의 [[다이얼로그 루트 노드]]에 부착하여 원하는 다이얼로그를 구현 할 수 있습니다.

다이얼로그 노드 부착하기​

그래프의 빈 공간에서 우클릭을 하여 노드 생성 메뉴를 열고 노드를 추가 할 수 있습니다. 예시에선 [[모놀로그 노드]]를 하나 생성하여 루트 노드의 start 핀과 연결하겠습니다

모놀로그 노드는 다수의 대화 참가자들끼리의 선형적인 대사 한 줄을 표현하는 기본 노드입니다.

다이얼로그 노드에 대사 입력하기​

우리는 여기서 간단하게 이 모놀로그 노드의 가운데에 있는 텍스트 입력 칸에 기본적인 대사를 넣어 보겠습니다.

![[script.png]]

대사를 추가했습니다. 이제 이 다이얼로그 매니저가 실행되어 루트 노드부터 시작해 진행하다가 이 노드를 만난다면 [[다이얼로그 위젯 베이스]]는 화면에 Hello, World of Simple Dialogue System 이라는 메세지를 화면에 표시 할 것입니다.

[[다이얼로그 노드 베이스 콘텍스트]] 기반의 노드들은 이런식으로 노드 중앙에 대사를 기입 할 수 있는 텍스트 박스가 뚫려있고, 이는 곧 다이얼로그 노드 베이스 콘텍스트 클래스의 Context Text 프로퍼티에 저장됩니다. 이 프로퍼티가 바로 다이얼로그 위젯이 다이얼로그 재생 도중 이 노드를 만났을 때 실행 할 대사가 됩니다. 자세한 내용은 [[다이얼로그 노드 베이스 콘텍스트]] 문서를 참고 하십시오.

지금까지 만든 다이얼로그를 재생하면 다음과 같은 결과물을 얻습니다:

다이얼로그 노드에 입력한 대사가 정상적으로 표시되는 것을 알 수 있습니다.

다이얼로그 참가자 지정하기​

다이얼로그 노드에 입력한 대사가 정상적으로 표시는 되지만, 대사를 말하는 캐릭터의 이름이 표시되어야 할 부분이 비워져 있어 해당 대사를 누가 말하고 있는지 알 수 없습니다. 여기서 대사의 참가자를 지정하여 이 대사를 누가 말하고 있는지, 그리고 누가 듣고 있는지를 정해봅시다.

![[participant.png]]

다이얼로그 노드를 선택했을 때 뜨는 노드의 프로퍼티 디테일 탭을 통해서 해당 노드의 대사에 대한 참가자를 지정할 수 있습니다.

Participant 섹션에 Speakers 프로퍼티와 Listeners 라는 프로퍼티가 존재하는 것을 볼 수 있습니다. 말그대로 각각 해당 대사를 말하는 캐릭터들과 듣는 캐릭터들을 나타내는 프로퍼티입니다.

우린 이 프로퍼티들에게 이 대사에 참여하게 하고 싶은 캐릭터들이 지닌 [[스피치버블 핸들 컴포넌트]]의 IDName 프로퍼티의 값을 넣어 해당 캐릭터가 대화에 참여하도록 할 수 있습니다.

[!note] 정확히는, 액터 하나가 여러가지의 [[스피치버블 핸들 컴포넌트]]를 지닐수 있고, 이들 모두 각기 다른 IDName을 지니게 하여 액터 하나당 여러 다이얼로그 참가자가 존재 할 수 있으므로 (이러한 특징은 히드라, 케로베로스, 다중인격 캐릭터처럼 한 액터에 여러 참가자가 존재하는 캐릭터를 제작하기 좋습니다), 캐릭터 액터가 대화에 참여하기 보다는 해당 핸들 컴포넌트가 대화에 참여한다고 보는 것이 바람직합니다.

자세한 내용은 [[스피치버블 핸들 컴포넌트]] 문서를 참고하십시오.

우선 해당 프로퍼티들을 채워주기 전에, 월드 상에 존재하는 우리 캐릭터들에게 [[스피치버블 핸들 컴포넌트]]를 부착하여 다이얼로그와 관련된 데이터를 추가하여 줍시다.

먼저, 캐릭터를 월드에 넣어보았습니다.

왼쪽에 있는 캐릭터는 Jack 이고 오른쪽에 있는 캐릭터는 Manny입니다. 우리에게 익숙한 언리얼 기본 스켈레탈의 모습을 한 친구들이죠.

캐릭터를 넣었으니 이제 스피치버블 핸들 컴포넌트를 부착한 뒤, 컴포넌트의 디테일 탭에 가보면 Display Name 이라는 프로퍼티가 있는 것을 볼 수 있습니다. 이 이름이 바로 다이얼로그 위젯 상에서 이 캐릭터를 나타내는 핸들 컴포넌트가 발화자로 지정되었을 때 표시되는 이름을 의미합니다.

이 프로퍼티에 각각의 캐릭터 이름을 넣어봅시다. Jack과 Manny를 각각 넣었습니다.

이제 그 아래에 있는 ID Name 이라는 프로퍼티에 각각 캐릭터들을 식별 하는 데 사용할 고유 ID를 넣어줍시다. 첫 번째 그리고 두 번째 튜토리얼 NPC라는 의미에서 NPC_Tutorial_1 , NPC_Tutorial_2를 각각 넣어 주겠습니다.

그럼 이제 다시 다이얼로그 매니저로 돌아와서, Speakers 배열에 엘리먼트를 추가하여 Participant Name 에 NPC_Tutorial_1을, Listeners 배열에 엘리먼트를 추가하여 Participant Name 에 NPC_Tutorial_2를 넣어줍시다.

그리고 한번 실행해보면, NPC_Tutorial_1 IDName을 지닌 Jack이 대사를 말하는 걸 볼 수 있고, NPC_Tutorial_2 IDName을 지닌 Manny이 대사를 듣고 있는 것을 볼 수 있습니다.

이 때 대사를 말하는 캐릭터의 이름이 Jack으로, 우리가 지정한 NPC_Tutorial_1 IDName를 지닌 핸들 컴포넌트의 Display Name이 표시 되는 것을 볼 수 있습니다.

여기서 주의할 점은 Display Name은 말그대로 표시되는 이름을 지정하는 용도로의 사용만 가능하고, 캐릭터를 식별하거나 특정하는 데에는 사용 할 수 없습니다. 가령, 지금처럼 캐릭터를 나타내는 핸들 컴포넌트들이 Manny와 Jack 처럼 서로 다른 Display Name을 가진 경우가 아닌 Enemy 혹은 ??? 처럼 (정체를 알 수 없는 캐릭터를 표현 할 때처럼) 완전히 똑같은 Display Name을 가지는 경우가 있을 수 있습니다.

이런 경우를 대비하여 플러그인은 ID Name 프로퍼티를 활용하여 캐릭터를 식별 하는 것입니다. 이를 이용하면 같은 표시 이름을 가지더라도 유니크한 ID를 활용하여 캐릭터를 원활하게 처리 할 수 있습니다.

한번 테스트 용도로 두 캐릭터의 Display 이름을 Not Same Character 로 변경 하고, 두 캐릭터 모두 발화자로 넣어보겠습니다.

그러면 다음과 같이 똑같은 이름을 가진 다른 캐릭터 둘이 대사를 함께 말하는 걸 볼 수 있습니다.

다이얼로그 실행하기​

사실 여기까지 따라오셨다면 다이얼로그 매니저를 구성하는 기본적인 방법은 전부 배우셨다고 할 수 있습니다. 이제 가장 중요한 다이얼로그를 실행하는 방법에 대해서 알려드리겠습니다.

축하합니다! 방금 막 다이얼로그 제작 세계에 첫 발걸음을 떼셨습니다.

그러나 아직 보여드리고 싶은 멋있고 쉽고 강력한 기능이 한움큼이나 남았습니다.

남은 튜토리얼은 가장 쉽게 플러그인을 학습 할 수 있는 순서대로 구성 되어있으니, 여기서 그만두지 마시고 다음 튜토리얼도 따라해보며 플러그인의 기능을 학습 해보시길 권장 드립니다.

다음 튜토리얼은 [[컨디션 써보기]]입니다.

[!note] 이제 플러그인의 강력한 기능들을 하나 씩 배워봅시다. 다음 튜토리얼은 [[기본 노드들 써보기]]입니다.
