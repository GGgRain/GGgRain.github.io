기본 노드들 써보기

이 문서에서는 플러그인에 존재하는 기본 제공 노드 클래스들의 용법을 알아보고, 이를 통해 기본적인 스크립트 제작 방법을 학습합니다.

[!info] 이 문서에서는 각 노드 타입별로 추가로 설명해야하는 기능과 달라지는 기능에 대해서만 설명합니다. 공통적인 내용은 [[인라인 커맨드 써보기]], [[콘텍스트 텍스트 스타일링 하기]]처럼 개별 문서로서 기술합니다. 이 점 참고하십시오.

모놀로그 노드​

모놀로그 노드는 여러 발화자들간의 대사를 표현 할 때 사용 하는 노드입니다.

[[첫 다이얼로그 만들기]] 문서에서 설명하였 듯, 노드 가운데에 있는 텍스트 박스에 원하는 대사를 넣어준 뒤 발화자 혹은 청취자를 지정해주면 바로 사용이 가능합니다. 발화자와 청취자는 꼭 한명일 필요가 없고, 여러 캐릭터를 지정 할 수 있습니다.

![[Pasted image 20231125095931.png]]

실행 결과는 다음과 같습니다.

![[Pasted image 20231125102606.png]]

우리가 지정한 발화자들이 대사를 말하는 모습을 볼 수 있습니다.

셀렉트 노드​

셀렉트 노드는 질문과 그에 대한 대답을 표현 할 수 있는 노드입니다.

질문을 위한 텍스트와 플레이어가 선택 할 수 있는 대답을 추가 할 수 있습니다.

우선 노드를 만들고, 참가자를 지정 하고 기본적인 텍스트을 넣어줍니다.

![[Pasted image 20231125103125.png]]

그 후, 디테일 탭에서 Answers 프로퍼티에 새로운 엘리먼트를 추가하여 이 노드에 표시할 대답을 넣어줍니다.

![[Pasted image 20231125103205.png]]

그러면 노드상에서도 우리가 추가한 대답이 표시되는 걸 볼 수 있습니다.

![[Pasted image 20231125103223.png]]

각 대답들은 노드상에서 핀으로 구현되며, 해당 대답이 선택되었을 때 다음에 실행할 노드들을 부착 할 수 있습니다. 이를 통해 간단한 분기 제어가 가능합니다.

![[Pasted image 20231125105135.png]]

실행 결과는 다음과 같습니다.

![[Pasted image 20231125104930.png]]

질문 텍스트와 우리가 추가한 대답이 제대로 표시 되는 걸 볼 수 있습니다. 여기선 첫번째 대답을 눌러봅시다.

![[Pasted image 20231125105249.png]]

그러면 첫 번째 대답에 연결된 노드가 실행이 되는 걸 볼 수 있습니다.

또한 각각의 대답 옵션 버튼 별로 표시될 텍스트에 적용할 텍스트 스타일 테이블을 따로 지정 하거나, 다이얼로그 텍스트 홀더 위젯 클래스를 변경하여 색다른 버튼을 제작하거나, 콘텍스트 텍스트 바인딩을 추가할 수 있습니다.

예제에서는 첫 번째 대답 옵션의 텍스트 스타일 테이블을 변경하여 다른 폰트를 사용하는 텍스트 테이블을 넣었습니다.

![[Pasted image 20231125105418.png]]

그러면 첫 번째 옵션 버튼의 텍스트 스타일링이 변경되는 것을 볼 수 있습니다.

![[Pasted image 20231125112107.png]]

또한 셀렉트 노드만을 위한 [[다이얼로그 인라인 커맨드]]로 DI_StartSelect가 제공됩니다. 시그니처는 StartSelect로, 텍스트 내부의 원하는 지점에서 이를 호출함으로서 대답 선택지 버튼을 그 시점에서 호출 할 수 있습니다.

텍스트의 커맨드를 넣고 싶은 지점을 클릭하여 에딧 커서를 올린 뒤, 위에 Bind Command 버튼을 클릭하고 StartSelect를 찾아 누릅니다.

그러면 다음과 같이 StartSelect 커맨드가 바인딩 되었습니다.

[!Info] 다이얼로그 인라인 커맨드의 사용에 대한 자세한 설명은 [[인라인 커맨드 써보기]]문서에서 다룹니다. 일단은 그냥 그렇구나 하고 넘어가시고, 나중에 해당 문서의 학습을 마무리 한 뒤 다시 오셔서 읽어보시길 권장합니다.

StartSelect 커맨드를 텍스트 중간에 넣었을때의 실행 결과입니다. 우리가 커맨드를 추가한 지점에 도달하면 대답 버튼들이 표시 되는 걸 볼 수 있습니다.

만약 텍스트 내에 해당 인라인 커맨드가 바인드되지 않았다면, should automatically append StartSelect at the end 프로퍼티가 true일 때 텍스트의 끝에 자동으로 이 명령이 추가되어 텍스트가 전부 표시되고 나서 자동으로 대답 버튼들이 표시됩니다.

[!Warning] 만약 해당 프로퍼티가 false 라면 버튼이 정상적으로 표시 되지 않고, 다이얼로그도 정상적으로 다음으로 넘어가지지 않습니다. 이 점 유의하십시오.

랜덤 노드​

아웃풋 핀에 연결 된 노드를 지정된 랜덤 로직에 따라 선택 및 실행시키는 노드입니다.

랜덤 로직은 랜덤 노드마다 지정이 가능한 랜덤마이저 인스턴스에 작성된 로직으로 처리됩니다.

노드를 추가한 뒤 앞에 노드와 연결 하고 난 뒤, 랜덤으로 재생하고 싶은 노드들을 랜덤 노드의 이웃풋 핀에 연결 해 주었습니다.

그 뒤 랜덤마이저 인스턴스를 선택해줘야합니다. 물론 기본 제공 랜더마이저 인스턴스를 사용 할 수도 있지만 이번엔 새로 만들어봅시다.

새 블루프린트 만들기 버튼을 눌러 다이얼로그 랜더마이저 인스턴스를 선택합니다.

만들어진 블루프린트에 들어가 CalculateChance 함수를 오버라이드 합니다.

이 함수가 바로 각 노드를 실행시킬 함수를 계산하고 리턴하는 부분은 담당합니다. 랜덤 노드는

이제 이 인스턴스를 아까 만든 랜덤 노드에 넣어주고 테스트 해봅시다.

그러면 우리가 만든 로직에 따라서 랜덤하게 다음 노드가 실행 되는 걸 볼 수 있습니다.

익스큐트 노드​

익스큐트 노드는 실행시에 노드에 등록된 모든 이벤트를 순차적으로 실행시키는 노드입니다.

익스큐트 노드를 만든 뒤에, 노드의 디테일 탭에서 Events to provoke 프로퍼티에 이벤트를 넣어봅시다.

실제로 실행 시켜보면, 해당 노드를 만났을 때 익스큐트 노드에 등록시킨 이벤트들이 호출 되는 것을 볼 수 있습니다.

브랜치 노드​

브랜치 노드는 주어진 다이얼로그 컨디션의 테스트 결과에 따라 분기를 제어해주는 노드입니다.

브랜치 노드를 만든 뒤에, 노드의 디테일 탭에서 BranchConditions 프로퍼티에 컨디션을 넣어봅시다.

실제로 실행 시켜보면, 해당 노드에 등록된 다이얼로그 컨디션들의 실행 결과에 따라, 테스트에 통과하면 True 핀에 붙은 노드들을, 아니면 False 핀에 붙은 노드들을 실행 시키는 것을 볼 수 있습니다.
