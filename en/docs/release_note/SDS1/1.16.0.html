1.16.0

Node Renaming​

노드 리네임 기능이 추가되었습니다. 그래프상에서 그래프 노드의 이름을 더블 클릭하여서 수정 할 수 있습니다. 이제 모든 다이얼로그 노드는 에디터 상에서 클래스 명이 아니라 노드의 실제 오브젝트 이름으로 표시됩니다.

Start Dialogue From Specific Node​

Simple Dialogue Subsystem의 Start Dialogue 함수에 NodeNameToStartThisDialogueFrom 프로퍼티가 추가 됨. 노드 이름을 지정함으로서 다른 노드를 건너뛰고 해당 노드부터 다이얼로그를 실행 할 수 있습니다. 값을 지정하지 않는 경우나 노드 이름이 올바르지 않는 경우 다이얼로그의 첫 부분 부터 재생을 시작합니다.

  • 추후 업데이트에서 노드가 존재하는지 여부를 검사하는 함수를 추가할 예정입니다.

Search & Replace​

다이얼로그 매니저에 Search & Replace 기능이 추가 되었습니다. Ctrl + F 로 Search 탭을 열 수 있고, Ctrl + H로 Replace 탭을 열 수 있습니다.

차차 다른 부분도 지원 되겠지만 현재는 Text / Node / Subnode의 검색만 지원하며, Replace 기능은 Text만 지원됩니다.

Dialogue Management Tab Reworked​

Placeholder 및 resolver용 기능 모음집에 불과했지만 이제는 Search & Replace 탭을 탑재하여 모든 다이얼로그를 한눈에, 일괄적으로 수정 할 수 있게 변경 되었습니다. 많은 다이얼로그 매니저 상에서 일괄적으로 변경해야하는 텍스트나 (예: 캐릭터 이름이 바뀌었다던가...) 스크립트의 변경이 매우 용이해졌습니다.

New Formatting Method​

이제 Context Finalizer (Context Binding)가 플러그인에서 제거되었습니다. 이는 Context Finalizer 가 엔진의 기본적인 에셋 처리 구조와 상반되는 구조 였고, 이를 수정 하기 보다는 아예 Fragment를 사용하는 체계로 넘어가는 것이 훨씬 유리했기 때문입니다.

이제 플러그인은 Fragment를 활용하여 포매팅을 처리합니다.

Fragment가 Context Finalizer에 비해서 가지는 강점은 다음과 같습니다.

  1. 용량이 적게 듭니다. Context Finalizer에 비해서 기본 용량이 적으며 같은 포매팅 기능이더라도 오브젝트 개수만큼 그래프를 가져 하나마다 전부 용량을 풀로 차지하던 Context Finalizer와 다르게 Fragment는 클래스 하나만큼의 용량이 전부이며 오브젝트 인스턴스 또한 별로 용량을 안먹습니다.
  2. 재활용이 가능합니다. 아이템 가격같은 자주 사용되는 포매터는 양식을 정해서 포매팅을 전담하는 Fragment를 작성해두면 매번 귀찮게 Context Finalizer를 새로 쓸 필요 없이 그냥 가져다가 붙혀서 플러그 앤 플레이로 사용 가능합니다.
  3. 조합이 가능합니다. 하나의 텍스트의 여러개의 포매터를 붙혀서 원하는 대로 포매팅이 가능하다는 뜻입니다. 가령 한 텍스트 내부에 아이템 가격과 스킬 이름을 포매팅해야하는 경우, 게임의 아이템 가격을 처리하는 포매터와 스킬 이름을 처리하는 포매터를 만들어서 둘 다 붙혀주면 각각 필요한 부분을 포매팅하므로, 기존 Context Finalizer에서 하였듯 같은 포매팅 로직을 합쳐서 두번 작성할 필요가 없어졌습니다.

Pin Related​

1.15.x 버전에 추가 되었던 Pin 기능의 아이콘 기능을 위해서 아이콘의 스케일을 조절 할 수 있는 프로퍼티가 추가 되었습니다.

또한 핀의 라벨에 리치텍스트를 사용 할 수 있도록 프로퍼티가 추가 되었으며, 커스텀 노드에서 핀 스트럭쳐를 업데이틀 할 때 지정하여 원하는 텍스트 스타일을 사용 할 수 있도록 변경 되었습니다.

Optimization​

플러그인이 C++ 빌드시에 Unity Build로 인해 숨어있던 Dependency 문제를 대량 해결하였음. 그리고 이에 따라 UBT의 오버헤딩 이슈를 처리하여 C++ 빌드 속도를 비약적으로 향상했습니다.

이에 따라 플러그인 자체를 변경하여서 C++ 빌드로 사용하는 경우에 빌드 시간에 큰 최적화가 이루어졌습니다.

Dialogue Text Holder Widget Changed​

이제 자식 레퍼런스를 디자이너 탭에서 이름을 통해서 받아오는게 아니라, 직접 함수를 통해서 제공해야합니다. Construct 이벤트에서 SetMainTextBlock 함수를 이용해 사용할 텍스트 블럭의 레퍼런스를 지정하십시오.

또한 Participant 스트럭쳐와 Context 스트럭쳐의 Override Text Holder Widget Class 기능에서 만약 오버라이드 할 클래스가 None이면 기본 클래스를 사용하는 것이 아니라 아예 위젯을 만들지 않는 것으로 변경되었습니다. 이름이나 텍스트를 표시하고 싶지 않을 때 도움이 됩니다.

Bug Fixes​

프레그먼트가 드래그 드랍으로 다른 노드에 붙었을 경우에 제대로 인스턴스가 초기화 되지 않아 에디터를 벗어나면 프래그먼트가 사라지는 버그가 수정되었습니다.

다이얼로그 위젯이 화면사이즈가 작을 때 끝자락에서 마구 요동치는 현상을 해결하였습니다.

다이얼로그 노드의 On Node Connection List Changed 함수가 빌드 버전에서 엔진 크래쉬를 야기하던 문제를 해결하였습니다.

Known Issues​

에디터 퍼포먼스가 살짝 떨어지는 것이 보고 되었습니다. 또한 에디터 코드 가독성 혹은 구조가 상당히 복잡해져서 추후 리펙토링이 살짝 필요해보입니다.
