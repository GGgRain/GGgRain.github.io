DialogueNode

[[다이얼로그 매니저]]에 추가하여 다이얼로그의 대사나 선택지, 브랜칭, 이벤트 호출, 흐름 제어, 특수 이벤트, 커스텀 액션를 표현하는 등 다이얼로그를 구성하는 각각의 흐름 섹션을 나타내는 노드를 의미합니다.

작동 원리​

[[다이얼로그 위젯 베이스]] 클래스는 [[다이얼로그 매니저]] 클래스를 이용해 다이얼로그를 표현하는데, 이때 매니저의 [[다이얼로그 루트 노드]]에 연결된 하위 다이얼로그 노드들을 스캔하여 테스트 및 실행하고, 한 노드가 종료 되면 그 노드에 연결 된 자식 노드를 스캔하여 컨디션을 테스트한 뒤 재생하는 방식으로 전체적인 흐름을 만듭니다.

다이얼로그 노드는 일종의 명령 및 데이터 집합으로서 작동하여, 다이얼로그 위젯이 해당 노드를 만났을때 실행할 로직과 초기화에 사용할 데이터를 가집니다.

이때 다이얼로그 노드를 새로 정의하여 기존의 작동 방식을 변경하여, 다이얼로그 위젯이 해당 노드를 만났을 때 어떤 로직을 실행할 지 덮어 써 아예 새로운 위젯을 화면에 표시하게 하거나, 아예 다른 노드로 흐름을 리다이렉트하는 등의 자유로운 활용이 가능하며, 이는 Simple Dialogue System의 유연성을 확보 시켜주는 핵심 설계 철학 중 하나입니다.

노드 기본(베이스) 클래스​

다이얼로그 노드는 크게 두가지 기본 클래스를 지닙니다. 각각의 기본 클래스는 [[다이얼로그 위젯 베이스]] 클래스와 긴밀하게 작용하는 다이얼로그의 핵심적인 기능 및 프로퍼티가 정의 되어있습니다. :

  • [[다이얼로그 노드 베이스]]
  • [[다이얼로그 노드 베이스 콘텍스트]]([[다이얼로그 노드 베이스]]에서 파생됨)

모든 종류의 다이얼로그 노드는 에디터상에서 그래프 노드로 표현 할 수 있습니다. 노드의 레이아웃 자체를 새로 정의 할 수는 없지만 에디터는 해당 노드가 가진 클래스의 가장 가까운 부모 클래스에게서 그래프 노드 레이아웃을 따와 새로운 자식 노드에게 적용시킵니다. 이로 인해 자신이 원하는 형태의 기능이나 노드 형태가 존재 할 경우, 이미 존재하는 기본 제공 노드 클래스에서 이를 따오는 것이 도움이 됩니다.

노드 우선도​

같은 핀에 연결된 각각의 다이얼로그 노드끼리는 우선도가 존재합니다.

노드가 연결된 핀에 또다른 노드를 연결하면 노드 오른쪽 구석에 숫자가 표시되는 것을 볼 수 있습니다.

더 낮은 숫자는 더 높은 우선도를 의미합니다. 다이얼로그가 재생될 때 더 높은 우선도를 가진 노드를 먼저 재생하려 시도합니다. 만약 그 노드가 실행이 불가능 할 경우([[다이얼로그 컨디션]] 테스트에서 실패한 경우) 그 다음으로 높은 우선도를 지닌 노드의 실행을 시도합니다. 만약 모든 노드가 실행에 실패하면 다이얼로그는 종료됩니다.

이를 활용하여 노드와 다이얼로그 컨디션을 바탕으로 다이얼로그의 분기를 제어 할 수 있습니다.
