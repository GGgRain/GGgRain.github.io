SpeechBubbleWidget_Handled

스피치버블 위젯 핸들드는 월드 및 스크린 기반 트랜스폼으로 다이얼로그 참가자 액터들을 따라다니며 렌더링 해주는 기능이 내장된 다이얼로그 위젯의 일종입니다.

렌더링 관련​

렌터링 모드 (렌더링 타입)​

스피치버블 위젯 핸들드는 게임에 사용되는 다이얼로그 위젯이 지원해야 할 렌더링 스타일 몇가지를 기본적으로 지원하며, 런타임중에 각각의 스타일로 절체 할 수 있습니다.

스피치버블 위젯 핸들드 클래스의 SetMode 함수에 의하여 렌더링 스타일 모드를 변경 할 수 있습니다. 아래는 각각의 모드에 대한 설명입니다. (EDialogueWidgetModeType 이넘 타입).

[!Info] 개발 초기에는 스피치버블 위젯 핸들드 클래스와 더불어 다이얼로그 위젯 핸들드 클래스라고 불리우는, 화면에 고정된 형태의 위젯 클래스가 따로 존재했으나, 업데이트를 걸쳐 스피치버블 위젯 핸들드 클래스가 해당 클래스의 기능을 흡수했습니다. SetMode 함수는 이런 변화의 일종으로 추가된 함수입니다.

WorldFollow​

다이얼로그 참가자들의 월드 트랜스폼을 따라 렌더링하는 모드입니다.

다이얼로그 위젯에서 현재 재생되고 있는 노드가 [[다이얼로그 노드 베이스 콘텍스트]] 기반의 노드일 시, 다이얼로그 참가자 중 발화자들의 트랜스폼의 평균 위치에 렌더링합니다.

WorldDynamic​

주어진 월드 트랜스폼의 위치에 위젯을 렌더링 하는 모드입니다.

SetRenderingTargetTransform 함수를 이용해 보간 목표 트랜스폼을 지정 할 수 있고 (레깅 기능이 꺼져있다면 해당 위치로 바로 렌더링됩니다.) RenderWidgetAtTransform 함수를 이용해 즉시 보간 없이 해당 트랜스폼으로 렌더링 위치를 바꿀 수 있습니다.

Screen​

주어진 스크린 상 비율 좌표에 위젯을 렌더링 하는 모드입니다.

비율 좌표는 스크린의 좌표를 0~1사이의 숫자로 표현한 좌표이며,

왼쪽 상단 코너를 (-1,-1), 정중앙을 (0,0), 오른쪽 하단 코너를 (1,1)로 취급합니다.

추가적으로 ScreenSpaceDepthOffset 라는 변수가 존재하는데, 이는 스크린 모드일 때 위젯이 화면상에서 얼마 만큼 떨어진 위치에 렌더링 되는지를 나타냅니다.

[!Warning] 모드 전환 시 자연스러운 보간을 위하여 적어도 300 이상의 값으로 설정해 두는 것을 추천드립니다.

SetRenderingTargetScreenCoordinate 함수를 이용해 보간 목표 좌표을 지정 할 수 있고 (레깅 기능이 꺼져있다면 해당 위치로 바로 렌더링됩니다.) RenderWidgetAtScreenCoordinate 함수를 이용해 즉시 보간 없이 해당 좌표로 렌더링 위치를 바꿀 수 있습니다.

추가 : 부드러운 모드 전환​

빌트인 애니메이션과 더욱 자연스러운 위젯 모드 전환을 지원합니다.

플러그인에 기본 제공되는 WB_SpeechBubble 위젯은 SetMode 함수로 모드가 변경 되었을때 이를 델리게이트로 감지해 위젯을 확장시키는 애니메이션과 축소시키는 애니메이션을 통해 스크린 모드일 시 화면 아래에 고정된 텍스트 박스로 위젯이 표시되도록 합니다. 추가로 넓어진 텍스트 박스 사이즈에 맞게 텍스트의 랩핑 로직도 자동으로 변경해줍니다.

이 기능을 원하지 않으신다면 Set Mode 전환시 애니메이션 재생을 하지 않도록 블루프린트에서 로직을 제거하시면 됩니다.

렌더링 클램핑​

위젯이 화면 밖으로 얼마만큼 나갈 수 있는지 지정 할 수 있는 기능이 내장되어 있습니다.

bShouldClampRenderLocation 변수가 true일 때 클램핑 기능이 활성화 되며, RenderLocationClampMargin 의 숫자에 따라 화면 안쪽으로 클램핑 되는 정도가 바뀝니다. 0일 때 가 화면 가장 가장자리까지 클램핑이 되며, 1일 때는 화면 정중앙에 고정됩니다.

렌더링 레깅​

위젯의 렌더링 트랜스폼이 변경 될 때, 새로운 트랜스폼과 기존 트랜스폼을 보간하는 기능입니다.

렌더링 모드가 WorldFollow 혹은 WorldDynamic일 때 bShouldApplyTransformLag 변수의 값이 true라면 월드 트랜스폼 변경에 보간이 적용됩니다. 이 때 TargetTransformInterpolationLagAmount의 값을 조정하여 보간 세기를 조절 할 수 있습니다.

렌더링 모드가 Screen일 때 bShouldApplyScreenCoordinateLag 변수의 값이 true라면 스크린 좌표 변경에 보간이 적용됩니다. 이 때 TargetScreenCoordinateInterpolationLagAmount의 값을 조정하여 보간 세기를 조절 할 수 있습니다.

이 보간 기능은 모드를 변경 할 때도 적용 되며, 각 모드 간 부드러운 전환을 지원하는 데에 사용 됩니다.

엣지 블러 기능​

위젯의 렌더링트랜스폼이 화면의 모서리 밖으로 나갈때 너무 많이 나갈시 자동으로 위젯의 투명도가 낮아지는 기능입니다.

하이드 블러 기능​

위젯과 플레이어 사이에 물체가 있을시 위젯의 투명도가 낮아지는 기능입니다.

어드벤스드 오파시티​

위젯의 오파시티 적용에 렉과 인터폴레이션을 넣어서 부드럽게 보간시키는 기능입니다.

스킵 오퍼레이션 인디케이터​

스킵 가능 여부를 표시하는 자그마한 위젯입니다.
