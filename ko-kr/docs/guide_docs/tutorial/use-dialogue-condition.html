다이얼로그 컨디션 써보기

다이얼로그 컨디션 만들기​

다이얼로그 컨디션을 우선 하나 만듭시다. 다이얼로그 매니저 에디터의 툴바에서 New Dialogue Condition 버튼을 클릭해줍시다.

이제 뜨는 옵션중에서 자신이 원하는 컨디션 종류를 선택하시면 됩니다.

[!Info] 각 컨디션 종류에 대한 설명은 [[다이얼로그 컨디션]] 문서을 참고하십시오.

예제에서는 Instanced Condition를 선택하여 인스턴스 다이얼로그 컨디션을 만들겠습니다.

다이얼로그 컨디션 에셋을 생성 한 뒤에 그 에셋에 부여할 컨디션 로직을 작성하는 것은 다음 함수를 오버라이드 하는 것에서 시작됩니다 :

C++ 자손 클래스의 경우 : virtual bool CheckCondition(class UDialogueWidgetBase* Widget, const TMap<FName, UActorComponent*>& ConversationParticipants);

블루프린트 에셋인 경우 :

![[Image/GettingStarted/Untitled/override.png]]

여기서 원하는 로직에 따라 만약 컨디션 체크에 성공 했다면 true를, 아니라면 false를 리턴시키면 됩니다.

우리가 만든 건 블루프린트 에셋이므로, 두번째 방법을 따라보겠습니다.

간단하게, 플레이어가 눈 앞에 보이는 버튼을 누르면 월드의 게임모드 인스턴스의 스코어 프로퍼티에 숫자가 1씩 올라가고, 우리가 지정한 스코어보다 높은 값일 때만 테스트에 통과하도록 만들겠습니다.

이때, 목표 스코어를 저장 할 프로퍼티를 만들고 퍼블릭으로 지정하여 다이얼로그 매니저 에디터 상에서 표시되도록 만들었습니다.

그럼 이제 매니저상의 노드에게 우리가 방금 만든 컨디션을 부착해봅시다.

노드를 클릭하고, 컨디션 섹션의 컨디션 프로퍼티에게 새로운 엘리먼트를 추가해봅시다.

이제 인스턴스드 컨디션 섹션에 우리가 방금 만든 컨디션을 선택합시다.

그러면 이런식으로 노드 인스턴스에 컨디션 인스턴스가 부착된 것을 볼 수 있습니다. 이제 이 컨디션에 우리가 원하는 스코어를 넣어줍시다. 예제에서는 3을 넣겠습니다.

그 후 한번 재생해보면 현재 스코어가 우리가 지정한 3보다 높을때만 노드가 실행되는 것을 볼 수 있습니다.

아직 끝이 아닙니다. 노드 우선도를 활용하여 이 노드가 실행에 실패했을때 재생하고 싶은 노드를 지정할 수 있습니다. 이를 통해 스코어가 지정 값보다 낮다는 메세지를 플레이어에게 표시하겠습니다.

컨디션을 부착한 노드가 연결된 핀에 또다른 노드를 연결합니다. 그러면 노드 오른쪽 구석에 숫자가 표시되는 것을 볼 수 있습니다.

더 낮은 숫자는 더 높은 우선도를 의미합니다. 다이얼로그가 재생될 때 더 높은 우선도를 가진 노드를 먼저 테스트 하고, 만약 그 노드가 테스트에서 실패하면 그 다음으로 높은 우선도를 지닌 노드를 테스트하여 실행을 시도합니다. 만약 모든 노드가 테스트에서 실패하면 다이얼로그는 종료됩니다.

즉, 이렇게 세팅을 해둔다면, 만약 스코어가 지정된 스코어보다 낮아 성공 메세지가 담긴 노드가 실패하면 자동적으로 더 낮은 우선도를 가진 실패 메세지 노드가 재생 될 것입니다.

이런 방식으로 다이얼로그 내에서 분기 제어가 가능합니다.

혹은 [[브랜치 노드]]를 활용하여 특정 컨디션의 성공 여부에 따라 분기 제어를 할 수도 있습니다.

만약 우리가 지금 만든 로직을 브랜치 노드로 대체한다면 다음과 같습니다 :

브랜치 노드에 우리가 방금 만든 컨디션을 넣고, 컨디션 테스트 성공시에는 성공 메세지를, 실패시에는 실패 메세지를 담은 노드를 실행시도록 연결 했습니다.

자세한 내용은 [[브랜치 노드]] 문서를 참고하십시오.

[!note] 분기제어도 멋있지만 실제로 게임플레이 로직을 실행시킨다면 더 멋있을 겁니다. 다음 튜토리얼은 [[이벤트 써보기]]입니다.
