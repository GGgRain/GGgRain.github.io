다이얼로그 이벤트 써보기

[!Info] 이 문서는 [[다이얼로그 이벤트]]의 구성보다는 기능의 사용법에 대해서 중점적으로 다룹니다. 다이얼로그 이벤트에 대한 자세한 내용은 [[다이얼로그 이벤트]] 문서를 참고하십시오.

다이얼로그 이벤트는 다이얼로그 상에서 호출가능한 블루프린트 로직입니다. 단순한 코스메틱 로직부터 게임에 깊게 관여하는 로직까지 원하는 로직을 마음껏 작성하고 사용할 수 있습니다. 이 문서에서는 이런 다이얼로그 이벤트를 어떻게 만들고 사용하는지 배워봅니다.

다이얼로그 이벤트 만들기​

다이얼로그 이벤트를 쓰려면 우선 하나 만들어야겠죠. 다이얼로그 매니저 에디터의 툴바에서 New Dialogue Event 버튼을 클릭해줍시다.

이제 뜨는 옵션중에서 자신이 원하는 이벤트 종류를 선택하시면 됩니다.

[!Info] 각 이벤트 종류에 대한 설명은 [[다이얼로그 이벤트]] 문서를 참고하십시오.

예제에서는 Instanced Event를 선택하여 인스턴스 다이얼로그 이벤트를 만들겠습니다.

다이얼로그 이벤트 에셋을 생성 한 뒤에 그 에셋에 부여할 이벤트 로직을 작성하는 것은 다음과 같은 함수를 오버라이드 하는 것에서 시작됩니다 :

C++ 자손 클래스의 경우 : virtual void ReceiveEventTrigger(class UDialogueWidgetBase* Widget, const TMap<FName, UActorComponent*>& ConversationParticipants);

블루프린트 에셋인 경우 :

![[override 1.png]]

여기에 우리가 원하는 이벤트 로직을 넣어주면 됩니다.

우리가 만든 건 블루프린트 에셋이므로, 두번째 방법을 따라보겠습니다.

간단하게, 원하는 파티클 에셋을 넣어주고 이를 플레이할 참가자 액터를 넣어주면 그 액터의 위치에서 파티클이 재생되도록 만들어주는 로직을 넣었습니다.

이때, 파티클 에셋과 참가자 액터의 이름을 지정할 프로퍼티를 만들고 퍼블릭으로 지정하여 다이얼로그 매니저 에디터 상에서 표시되도록 만들었습니다.

그럼 이제 매니저상의 노드에게 우리가 방금 만든 이벤트를 부착해봅시다.

노드를 클릭하고, 이벤트 섹션의 이벤트 프로퍼티에게 새로운 엘리먼트를 추가해봅시다. 이벤트의 키는 HappyParticle 로 하겠습니다

이제 인스턴스드 이벤트 섹션에 우리가 방금 만든 이벤트를 선택합시다.

그러면 이런식으로 노드 인스턴스에 이벤트 인스턴스가 부착된 것을 볼 수 있습니다. 이제 이 이벤트에 우리가 원하는 파티클과 참가자 이름을 넣어줍시다.

아직 끝난게 아닙니다. 다이얼로그 노드는 노드에 부착된 이벤트를 자동으로 호출해주지 않습니다.

우리가 부착한 이벤트를 호출하려면 다른 모종의 방법이 필요합니다. 여기에는 정말 다양한 방법이 활용가능한데, 프래그먼트나 인라인 커맨드를 활용하여 호출하거나 또 다른 이벤트가 다른 이벤트를 호출하도록 설계할 수도 있습니다. 완전히 외부에서 이벤트를 호출하도록 접근하는 방법도 가능합니다. (다른 액터가 위젯 레퍼런스를 받아서 노드에 접근하여 호출) 노드의 작동방식을 변경해 이벤트가 자동으로 실행되도록 만들 수도 있지요. ([[익스큐트 노드]]를 통해 기본 제공)

여기선 가장 보편적이고 자주 사용하는 방식인 인라인 커맨드를 활용한 방법을 알아봅시다. 인라인 커맨드를 통해 텍스트의 원하는 지점에 도달 했을 때 이벤트를 호출하는겁니다. 이 방법은 플러그인에서 기본 제공하는 DI_Event 커맨드를 활용하여 이뤄집니다.

텍스트에서 이벤트를 호출하길 원하는 지점을 클릭하여 텍스트 에딧 커서를 올린 뒤, Bind Command 버튼을 누른 뒤 Event를 선택해줍니다.

인라인 커맨드가 넣어졌다면, 인라인 커맨드를 클릭하여 우리가 방금 이 노드에 부착한 이벤트의 이벤트 키를 넣어줍시다.

그 후 한번 재생해보면 우리가 이벤트를 넣은 부분이 실행 될 때 다음과 같이 제대로 이벤트가 호출되는 걸 볼 수 있습니다.

[!note] 게임플레이 로직을 실행시키는 저희 플러그인 만의 가볍고 창의적인 방법이 존재합니다. 이 문서에서 잠깐 등장한 Event 커맨드를 비롯한 인라인 커맨드가 바로 그 주인공입니다. 다음 튜토리얼은 [[인라인 커맨드 써보기]]입니다.
