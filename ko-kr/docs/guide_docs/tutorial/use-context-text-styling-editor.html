콘텍스트 텍스트 스타일링 에디터 써보기

[!Info] 이 문서는 언리얼 엔진의 리치 텍스트 시스템을 이해하고 있다는 가정 하에 설명 됩니다. Simple Dialogue System의 텍스트 스타일링 시스템은 언리얼 엔진의 리치 텍스트 시스템을 가져와 사용하며, 플러그인이 제공하는 에디터는 이 리치 텍스트 시스템을 보다 쉽게 사용 할 수 있게 도와주는 것에 중점을 두고 있습니다. 리치 텍스트 시스템에 대한 이해는 다음 문서를 참고 하십시오. https://docs.unrealengine.com/4.27/en-US/InteractiveExperiences/UMG/UserGuide/UMGRichTextBlock/

콘텍스트 텍스트 에디터는 [[다이얼로그 노드 베이스 콘텍스트]] 기반의 노드의 Context Text 프로퍼티의 텍스트에 더욱 쉽게 데코레이션을 넣을 수 있게 해주는 에디터입니다.

이번에는 이 콘텍스트 텍스트 에디터를 사용하는 방법에 대해 알아 봅시다.

기본 구성​

[[다이얼로그 노드 베이스 콘텍스트]] 기반의 노드를 [[다이얼로그 매니저]]의 그래프에 추가해봅시다. 예제에서는 [[모놀로그 노드]]를 사용하겠습니다.

![[Image/GettingStarted/Untitled/Editor.png]]

그러면 Context Text를 수정할 수 있는 텍스트 박스 위에, 여러가지 추가적인 버튼들을 볼 수 있습니다. 그중에서 우리가 사용 할 것들은 첫번째 섹션에 있는 버튼들입니다.

![[Editor2.png]]

각 버튼에 대한 설명은 다음과 같습니다.

  1. 이 텍스트블럭에 적용될 텍스트스타일테이블이 다이얼로그 매니저의 기본에셋을 사용해야 할 지 아니면 선택한 에셋으로 오버라이드 해야할지 결정하는 체크박스
  2. (1)에서 설명할 체크박스를 체크했을 때 이 텍스트블럭에 오버라이드 할 텍스트스타일테이블 에셋을 고를 수 있는 버튼
  3. 텍스트 구간에 적용할 스타일을 고를수있는 콤보박스

![[editor3.png]] ![[editor4.png]]

또한 추가적으로 노드의 가장 오른쪽에 있는 Main Edit이라고 써진 버튼을 누르면 Raw Edit으로 변경되면서 편집 모드가 마크업 데코레이터를 전부 표시하는 직접 수정 모드로 변경됩니다. 마크업 데코레이터를 직접적으로 수정 하고 싶을 때 큰 도움이 됩니다. 자세한 내용은 아래 설명에서 차근차근 알아가봅시다.

그러면 각 버튼을 어떻게 사용하는지 같이 한번 배워봅시다.

콘텍스트 텍스트 에디터로 텍스트 스타일링 해보기​

우선, 아까의 모놀로그 노드에 대사를 입력해봅시다. 예제에선 "Context Text Editor Example References" 라는 텍스트를 넣어보겠습니다.

![[Image/GettingStarted/Untitled/Editor.png]]

입력이 끝났다면, 스타일링 하고 싶은 텍스트 부분을 마우스로 드래그하여 선택해봅시다.

![[edit1.png]]

그 후, 저기에 있는 Default 라고 기입된 콤보박스를 눌러봅시다. 아까 구성요소 파트에서 3번째로 설명한 버튼입니다.

![[edit2.png]]

그러면 이 다이얼로그에게 주어진 텍스트 스타일 테이블로부터 우리가 선택 할 수 있는 스타일링 목록이 다음과 같이 표시됩니다. 이제 여기서 원하는 스타일링을 선택해 봅시다. 예제에서는 Rainbow를 선택해보겠습니다.

![[edit3.png]]

그러면 다음처럼 아까 전에 선택한 텍스트 부분이 선택한 텍스트 스타일로 스타일링이 된 것을 볼 수 있습니다.

텍스트 스타일 작성하기​

기본 제공되는 텍스트 스타일도 물론 좋지만, 원하는 텍스트 스타일을 넣고 싶을 때가 있을 것입니다.

기본적으로 [[다이얼로그 매니저]] 상의 [[다이얼로그 노드 베이스 콘텍스트]]들은 다이얼로그 매니저의 기본 텍스트 스타일 테이블을 사용하여 스타일링 되도록 설정되어있습니다. 그러므로 일단 우리는 다이얼로그 매니저에 등록되어있는 텍스트 스타일 테이블을 수정하여 우리가 원하는 텍스트 스타일을 추가 해보겠습니다.

그러면 원하는 텍스트 스타일을 넣기 위해서, 텍스트 스타일 테이블로 한번 가봅시다.

![[edit5.png]]

그래프 상에 빈 곳을 클릭하면 노드 선택이 해제 되면서 이 다이얼로그 매니저 에셋에 대한 디테일 탭이 표시가 됩니다.

![[edit6.png]]

우리가 봐야 할 것은 Appearance 카테고리의 Default Text Style Table 프로퍼티입니다. 이 프로퍼티에 들어가 있는 텍스트 스타일 테이블이 노드에 기본적으로 사용 되는 것입니다. 저기에 들어가 있는 DefaultTextStyle 에셋을 더블클릭하여 수정해봅시다.

![[edit7.png]]

테이블 에셋을 열어보면 다음과 같은 각각의 스타일의 정의 대한 열이 테이블에 들어가 있습니다.

여기에 새로운 열을 추가하여 우리의 스타일을 만들어 봅시다.

![[edit9.png]]

간단하게 이름은 TutorialText 로 하고, 70 사이즈 Putty 폰트의 기본 제공 폰트 머터리얼을 넣어주고, 쉐도우를 (2,2) 만큼 오프셋을 넣어 주었습니다. 그 후 이 테이블을 저장 한 후, 다이얼로그 매니저 에디터를 껏다가 켜봅시다.

![[edit10.png]]

그런 뒤에 다시 텍스트 스타일링 콤보박스를 열어보면 정상적으로 우리가 추가한 텍스트 스타일이 표시 되는 걸 볼 수 있고,

![[edit11.png]]

이를 선택하면 우리가 정의한 텍스트 스타일로 스타일이 바뀌는 것을 볼 수 있습니다.

텍스트 스타일 테이블 오버라이드 하기​

가끔은 노드 별로 원하는 텍스트 스타일 테이블을 쓰고 싶을 수도 있을 것입니다. 그럴 때 사용 하는 것이 바로 텍스트 스타일 테이블 오버라이드 기능입니다.

저 위에 OVERRIDE라고 표시된 체크박스 버튼을 눌러봅시다.

![[Override 2.png]]

그러면 기존에 텍스트에 적용 되어있던 텍스트 스타일들이 전부 빠지고 빨간색의 기본 NULL 텍스트 스타일로 전부 스타일링 되는 것을 볼 수 있습니다.이는 우리가 현재 기본적으로 적용 되던 다이얼로그 매니저의 텍스트 스타일 테이블을 적용 해제하고, 이 노드를 위한 오버라이드 텍스트 스타일 테이블을 스타일링에 사용하게끔 만들었기 때문입니다. 그러나 아직 우리는 어떤 텍스트 테이블을 사용할 것인지를 지정하지 않았고, 그 결과 이처럼 NULL 텍스트 스타일로 기본적으로 표시가 되는 것입니다.

그럼 이제 OVERRIDE 옆에 있는 오버라이드 텍스트 스타일 테이블 셀렉터를 통해 새로운 텍스트 스타일 테이블을 넣어봅시다. 저기 Select Text Style Table이라고 써진 버튼을 클릭하면...

![[Style.png]]

우리가 사용 가능한 텍스트 스타일 테이블이 표시 되는 것을 볼 수 있습니다. 여기서 이제 새롭게 사용할 텍스트 스타일 테이블을 넣어줍시다. 예제에선 그냥 기본 제공 테이블을 넣어보겠습니다.

![[edit12.png]]

그러면 정상적으로 텍스트가 스타일링 되는 것을 볼 수 있습니다.

[!Info] 예제 사진을 찍을 때 TutorialText 스타일을 이 사진을 찍기 전에 실수로 스타일을 지워버려서 TutorialText 스타일이 제대로 표시되지 않았습니다.... 만약 정상적으로 이 튜토리얼을 따라 오셨다면 아까 위에서 우리가 넣은 스타일이 표시가 됩니다.

직접 수정 모드 알아보기​

간혹 가다가 플러그인의 오류 혹은 사용상의 실수로 텍스트 스타일 데코레이터가 꼬이는 경우가 생길 수 있습니다. 혹은 아예 직접 텍스트 스타일링을 지정해주고 싶은 경우 테코레이터의 텍스트를 보고 싶을 수 있을 것 입니다. 이럴 때 저기의 Main Edit 버튼을 눌러 보시면 됩니다.

![[edit13.png]]

그러면 리치 텍스트의 스타일링 마크업 데코레이터가 전부 텍스트의 형태로 표시 되는 것을 볼 수 있습니다. 여기서 원하기는 데로 텍스트를 수정해 원하는 스타일로 변경하시면 됩니다.

Main Edit 버튼을 다시 클릭하는 것으로 직접 수정 모드를 종료 할 수 있습니다.

![[visi.png]]

만약 매번 버튼을 눌러서 직접 수정 모드로 넘어가는 게 번거로우시다면 툴바의 Visibility 섹션에서 Raw Context Text Editor Display Type을 In a row로 변경 해보십시오.

![[Edit14.png]]

그러면 아예 직접 입력 모드 텍스트 블럭이 노드 하단에 상시 표시 됩니다.

또한 Don't display 옵션을 선택하여서 아예 직접 입력 모드 전환 버튼을 숨겨버릴 수도 있습니다.

[!note] 텍스트 스타일링 뿐만 아니라 텍스트 자체도 변조가 가능합니다. 다음 튜토리얼은 [[콘텍스트 텍스트 바인딩 사용하기]]입니다.
